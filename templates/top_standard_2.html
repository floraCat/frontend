    规范目的

      为提高团队协作效率, 便于后台人员添加功能及前端后期优化维护,前端开发人员必须按本文档规范进行前端页面开发； 本文档如有不对或者不合理的地方请及时提出, 经讨论决定后方可更改。

    基本准则
    • 符合web标准
    • 结构/表现/行为 分离
    • 语义化html
    • 代码书写简洁明了有序
    • 测试需兼容Firefox,Chrome,Opera,IE(ie8及以上版本)浏览器;
    • 尽可能减小服务器负载, 保证最快的解析速度。
    前端架构

    html

    • 每个站点都应该有一个整站链接目录页（统一命名为__enter.html）
    • 整站通用的模块（如头部、底部等）需独立出来，各页面再通过java来引入

    css

    • 使用less预编译css

    javascript

    • 使用requirejs技术模块化引入脚本文件
    • 脚本一般基于jQuery库来书写：pc端使用1.10.2版本，移动端使用2.2.0版本
    • 表单都需要做前端验证
    • 每个页面所有变量和函数都放到一个全局对象中

    辅助工具

    • 使用Git版本控制工具来管理网站
    • 使用Gulp构建工具自动化打包网站

    css优化
  1. 样式都用class设置，慎用id
  2. 相同或类似的模块尽量把代码整合重用起来
  3. css慎用expression表达式
  4. 尽量使用css选择器（以防止属性设置冲突，从而减少类名设置，保持结构清晰）
  5. 如多个元素一并设置，元素之间换一行
  6. 尽量减少不必要的嵌套层级

    js基本设置
  1. 传统js脚本的编写，要求用下面的模板进行包裹：
    2. (function($){ 'use strict'; // code... }(jQuery));
    js格式
  1. 每句语句后的';'不能省略
  2. if、while、for、do语句的执行体总是用"{"和"}"括起来，即使在其结构体内只有一条语句
  3. 应该优先使用单引号(‘)，因为字符串可能包含带有双引号(“)的HTML代码
  4. 总是使用var声明变量，且要求变量集中在函数开头声明, 避免全局变量的污染
    js优化
  1. js用data-*作为选择器，不用class & id
  2. 使用闭包时防止循环引用
  3. 尽量避免使用存在兼容性及消耗资源的方法或属性, 比如eval() & innerText
  4. 避免对象上使用with()
  5. 避免数组使用for in
    js 注释
  1. 注释尽量详尽且到位
  2. 需写明所属页面或栏目
  3. 类等脚本文件需添加头描述
