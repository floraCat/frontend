'基本规范'全部展开
    规范目的

      为提高团队协作效率, 便于后台人员添加功能及前端后期优化维护,前端开发人员必须按本文档规范进行前端页面的开发； 本文档如有不对或者不合理的地方请及时提出, 经讨论决定后方可更改。

    基本准则
    • 符合web标准；
    • 结构/表现/行为 分离；
    • 语义化html；
    • 代码书写简洁明了有序；
    • 测试需兼容Firefox,Chrome,Opera,IE(ie8及以上版本)浏览器;
    • 尽可能减小服务器负载, 保证最快的解析速度。
    文件命名原则
    • 命名优先考虑对应英文，可使用常规缩写形式，如英文太生僻或复杂再考虑拼音
    • 命名尽可能使得你自己和工作组的每一个成员能够理解每一个文件的意义；
    • 同一类的文件使用一样的前缀，使得同类文件都能排列在一起，如： shopping //商城首页 shopping-cart //购物车 shopping-order //订购清单
    图片规范
    案例图片
    • 案例图片指前端页面用img元素调用的图片，一般为后台可配置的图片
    • 存放路径：项目根目录的/images文件夹内
    • 在保证视觉效果的情况下选择最小的图片格式与图片质量, 以减少加载时间
    • 非首页轮播大图外的图片大小尽量限定在 70K 左右或更小
    • 图片格式仅限于gif || png || jpg
    背景图片
    • 背景图片指样式中使用background属性设置的图片，一般为比较少改动的背景图或小图标
    • 存放路径：项目根目录的/css/imgs文件夹内
    • 项目发布前背景图需分类合并，以减小http请求
    • 图片格式仅限于gif || png || jpg
    • 透明背景的图标用png24格式；非透明背景的图标如图形扁平色彩单一的用png8格式，否则用jpg格式

    html结构
    html基本结构
  1. 必须有文档类型声明<!DOCTYPE HTML>，且在文档的第一行，位于html标签之前
  2. 使用utf-8编码
  3. 样式代码或外调style文件请在head元素中引入，如下： <head> <link rel="stylesheet" href="xxx.css" /> <style> ... //不建议直接在html文件内设置css样式 </style> </head>
  4. 脚本代码或外调的js文件在head元素中或在body元素内部的最后引入，如下： <head> <script src="xxx.js"></script> <script> ... </script> </head> <body> html code here ... <script src="xxx.js"></script> <script> ... </script> </body>
  5. 使用退格键缩进代码
    6. html参考模板
  6. 模板参看：html基本结构此文档内容说明： - 文档中的common.css为项目共用的样式表，包含重置样式、头部底部等网站共用模块的样式 - 文档中的iconfig.js为项目通用路径配置 - 项目使用了requirejs技术来模块化管理脚本代码，文档中的rconfig.js为requirejs的配置文件
    html标签
    html标签格式
  1. 不能省略可选的结束标签，如：<div>的结束标签</div>不能省略
    2. html标签使用
  2. h1 - 用于网页主题，一个页面只有一个
  3. h2 - 用于大栏目标题 or 楼层标题 等
  4. h3 - 用于小栏目标题
  5. h4 - 用于列表每一条目的标题
  6. h5,h6 - 用于除以上情况的其他如强调加粗等情况
  7. ul+li - 列表最佳元素组合
  8. p - 用于描述简介等文本
  9. span - 用于搭配其他元素用以布局标签等，尽量不要用于外容器
    10. html标签优化
  10. 页面元信息的title元素不可缺少
  11. 表单控件的标签文本使用label元素包裹，或使用其for属性确保点击标签文本时能激活对应控件，如： <label>check-1:<input type="checkbox" /></label> <lable for="check2">check-2:</lable><input type="checkbox" id="check2" />
    html属性
    html属性格式
  1. 标签属性值使用双引号，不要使用单引号
  2. ID属性在一个页面内是唯一的
  3. 外调文件如果是默认类型时不需要声明type，如： <script src="xxx.js"></script> <link rel="stylesheet" href="xxx.css" />
  4. 一般情况下不使用内置style
  5. 合理设置链接的target属性，如：target='_blank'
    6. html属性优化
  6. a标签如无链接时其属性href值填充':javascript:;'
  7. img标签需要加alt属性 (无意义的装饰性图标除外)
  8. a链接如锚文本不足以描述此链接，需要加title属性，如： <a title="力德网站" href="#">点击这里</a>
  9. 页面的关键字和描述（即元信息name为keywords/description）在发布前应添加合适内容
    html文本
  1. 文本内尽量避免使用实体,如：空格实体'&nbsp'
    html注释
  1. 网页每个模块都应该注释其栏目名称或简短说明,格式如下： <!--头部--> ... <!--/头部-->

  1. 模板参看：html相关命名参考

  5. 设置项目中常用样式的属性组合

    css格式
  1. 值为0时后面不加单位
  2. 颜色值都用小写
  3. 当属性值需要加引号时，请使用双引号，如： font-family:"Microsoft Yahei",arial,Tahoma,sans-serif; background:url("xxx.png"); content:"xxx";
  4. 每个样式属性后的分号";"不能省略
  5. 属性值尽量使用缩写，如： border:0 10px; margin:10px 15px 20px; padding:10px; color:#efg; background:#eee url(xxx.png) no-repeat center right;
  6. 需注释所属栏目或模块标题，跟html对应模块标题一致; 多个模块标题用'&'分隔，格式如下： /*案件列表 & 新闻列表*/ .p-list-pics{...}
    css属性声明顺序
  1. 模板参看：css属性声明顺序
    css优化
  1. 样式都用class设置，慎用id
  2. 相同或类似的模块尽量把代码整合重用起来
  3. css慎用expression表达式
  4. 基于减少类名设置、防止属性设置冲突、保持结构清晰的目的，尽量使用css选择器，如： ul > li{...} input[type=text]{...}
  5. 如多个元素一并设置，元素之间换一行，如： a:hover, a.on{...}
  6. 选择器尽量减少不必要的嵌套层级，如： 一个外容器类名为p-list的模块内只有一个ul和同级的li，设置li样式时用.p-list .li{...}而不用.p-list ul li{...}

'js规范'全部展开
    js基本设置
  1. 传统js脚本的编写，要求使用下面的模板进行包裹： (function($){ 'use strict'; // code... }(jQuery));
  2. requirejs脚本的编写，要求使用下面的格式： define([ "jquery", "angular" ], function ($, angular) { 'use strict'; // code... });
    js格式
  1. 每句语句后的';'不能省略
  2. if、while、for、do语句的执行体总是用"{"和"}"括起来，即使在其结构体内只有一条语句
  3. 代码编写时应该优先使用单引号(‘)，防止字符串可能包含带有双引号(“)的HTML代码
  4. 总是使用var声明变量，避免全局变量的污染
    js优化
  1. 使用闭包时防止循环引用
  2. 尽量避免使用存在兼容性及消耗资源的方法或属性, 比如eval() & innerText
  3. 避免对象上使用with()
  4. 避免数组使用for in
    js注释
  1. 脚本模块需写明所属页面或栏目，注释必要时尽量详尽且到位
  2. 插件等独立脚本文件需添加详细的头描述，格式参考如下： /* * 简述 * * 功能详细描述 * * @param <String> arg1 参数1 * @param <Number> arg2 参数2，默认为0 * @return <Boolean> 看xxx是否成功 */ +(function(){ 'use strict'; var pluginName = "xxx"; ... })();
