    规范目的

      为提高团队协作效率, 便于后台人员添加功能及前端后期优化维护,前端开发人员必须按本文档规范进行前端页面开发； 本文档如有不对或者不合理的地方请及时提出, 经讨论决定后方可更改。

    基本准则
    • 符合web标准
    • 结构/表现/行为 分离
    • 语义化html
    • 代码书写简洁明了有序
    • 测试需兼容Firefox,Chrome,Opera,IE(ie8及以上版本)浏览器;
    • 尽可能减小服务器负载, 保证最快的解析速度。
    前端架构

    html

    • 每个站点都应该有一个整站链接目录页（统一命名为__enter.html）
    • 整站通用的模块（如头部、底部等）需独立出来，各页面再通过java来引入

    css

    • 使用less预编译css

    javascript

    • 使用requirejs技术模块化引入脚本文件
    • 脚本一般基于jQuery库来书写：pc端使用1.10.2版本，移动端使用2.2.0版本
    • 表单都需要做前端验证
    • 每个页面所有变量和函数都放到一个全局对象中

    辅助工具

    • 使用Git版本控制工具来管理网站
    • 使用Gulp构建工具自动化打包网站

    html标签
    html标签格式
  1. 不能省略可选的结束标签
    2. html标签使用
  2. h1 - 用于网页主题 / 一个页面只有一个
  3. h2 - 用于大栏目标题 or 楼层标题 等
  4. h3 - 用于小栏目标题
  5. h4 - 用于列表每一条目的标题
  6. h5,h6 - 用于除以上情况的其他如强调加粗等情况适当使用
  7. ul+li - 用于列表最佳搭配
  8. p - 用于描述文字
  9. span - 用于搭配其他元素用以布局标签等，尽量不要用于外容器
    10. html标签优化
  10. 页面title不可缺少
  11. 表单控件的标签文本使用label元素包裹，或使用其for属性确保点击标签文本时能激活对应控件
    html属性
    html属性格式
  1. 标签属性值使用双引号，不要使用单引号
  2. ID属性在一个页面内是唯一的
  3. 外调文件如果是默认类型时不需要声明type，如： <script src="xxx.js"></script> <link rel="stylesheet" href="xxx.css" />
  4. 一般情况下不使用内置style
  5. 合理设置链接的target属性，如：target='_blank'
    6. html属性优化
  6. a标签如无链接时其属性href值填充':javascript:;'
  7. img标签需要加alt属性 (装饰性的图片或图标除外)
  8. a链接如锚文本不足以描述此链接，需要加title属性，如： <a title="力德网站" href="#">点击这里</a>
  9. 页面元信息name为keywords/description等在发布前应添加合适内容
    html文本
  1. 文本内尽量避免使用实体，如空格实体' '
    html注释
  1. 网页每个模块都应该注释其栏目名称或简短说明,格式如下： <!--头部--> ... <!--/头部-->

    css格式
  1. 值为0时后面不加单位
  2. 颜色值都用小写
  3. content的属性值用单引号包含，不是双引号，如：content:'xxx';
  4. 每个样式属性后的分号";"不能省略
  5. 属性值尽量用缩写，如： border:0 10px; margin:10px 15px 20px; padding:10px; color:#ccc; color:#efg; background:#eee url(xxx.png) no-repeat center right;
  6. 需注释所属栏目或模块标题，跟html对应模块标题一致; 多个模块标题用'&'分隔，格式如下： /*案件列表 & 新闻列表*/ .p-list-pics{...}
    css属性声明顺序
  1. 模板参看：css属性声明顺序
    css优化
  1. 样式都用class设置，慎用id
  2. 相同或类似的模块尽量把代码整合重用起来
  3. css慎用expression表达式
  4. 尽量使用css选择器（以防止属性设置冲突，从而减少类名设置，保持结构清晰），如： ul > li{...} input[type=text]{...}
  5. 如多个选择器一并设置，元素之间换一行，如： a:hover, a.on{...}
  6. 尽量减少不必要的嵌套层级

'js规范'全部展开
    js基本设置
  1. 传统js脚本的编写，要求用下面的模板进行包裹： (function($){ 'use strict'; // code... }(jQuery));
    js格式
  1. 每句语句后的';'不能省略
  2. if、while、for、do语句的执行体总是用"{"和"}"括起来，即使在其结构体内只有一条语句
  3. 代码编写时应该优先使用单引号(‘)，防止字符串可能包含带有双引号(“)的HTML代码
  4. 总是使用var声明变量，避免全局变量的污染，且要求变量集中在函数开头声明
    js优化
  1. 使用闭包时防止循环引用
  2. 尽量避免使用存在兼容性及消耗资源的方法或属性, 比如eval() & innerText
  3. 避免对象上使用with()
  4. 避免数组使用for in
    js注释
  1. 脚本模块需写明所属页面或栏目，注释必要时尽量详尽且到位
  2. 插件等独立脚本文件需添加详细的头描述，格式如下： /* * 简述 * * 功能详细描述 * * @param <String> arg1 参数1 * @param <Number> arg2 参数2，默认为0 * @return <Boolean> 看xxx是否成功 */ function fooFunction (arg1, arg2) { }
